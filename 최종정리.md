## 1. 최종 판정

현재 프로젝트는 **구현 범위는 충분하지만 그대로 제출하기엔 아직 위험**합니다.

개선된 점:
- `classifier_박연진.ipynb`는 프로젝트 루트 자동 탐색을 사용합니다.
- `README.md`는 조교 실행 흐름, Python/torch 환경, Qwen2.5-3B 기본 모델, 모델 다운로드 링크를 안내합니다.
- `data/valid.json`과 `data/train.json` 질문 중복은 현재 **0/30**입니다.
- `chatbot.sh`는 벡터 DB 정합성 검사 실패 시 자동 재구축하도록 되어 있습니다.
- 문법 검증: `python -m compileall -q src scripts tests` → **PASS**.

남은 차단 리스크:
- `outputs/chat_output.json`은 현재 5개 모두 `[MOCK]` 응답입니다.
- `python scripts/check_vector_db.py` 결과 현재 `chroma_db`는 **불일치**입니다: `build_manifest.json` 없음, DB 158건 / `chunks.json` 205건.
- 발표자료, UI 작동 영상, 최종 `Termproject_박연진.zip`은 현재 폴더에서 확인되지 않았습니다.
- 현재 로컬 환경에서 `pytest -q`는 `chromadb`, `sentence_transformers` 미설치로 실패했습니다.
- `src/rag/pipeline.py`는 GPU 0 강제 배치(`device_map={"": 0}`)와 4-bit bitsandbytes를 사용하므로, GPU 없는 조교 노트북 로컬 실행은 위험합니다.

**권고:** 최종 제출 전 `REQUEST CHANGES`. 코드 구현보다 실행 재현성, 출력 갱신, 제출 패키징을 마무리해야 합니다.

## 6. RALPLAN — 원인과 해결 계획

### Principles

1. PDF 요구사항을 최종 기준으로 삼습니다.
2. 조교 실행 진입점인 `classifier_박연진.ipynb`와 `chatbot.sh`를 최우선 검증합니다.
3. 제출 전 출력 파일에는 MOCK, 오래된 DB, 미래 날짜 오류가 남지 않아야 합니다.
4. 조교 환경에서 재현 가능한 실행 방법을 README/발표자료에 명확히 남깁니다.
5. 삭제는 원본 보존 후 제출 패키지에서만 제외합니다.

### Decision Drivers

1. 실행 재현성
2. 출력 JSON 형식과 실제 응답 품질
3. 제출물 완성도

### 추천 선택지

**Option A — 최소 제출 안정화**를 추천합니다.

할 일:
1. 벡터 DB 재구축
2. `chatbot.sh` clean-run
3. 출력 JSON 갱신
4. 발표자료/영상/ZIP 제작
5. 불필요 파일 제출 ZIP에서 제외

Option B처럼 Task1 분류 모델을 Task2 라우터에 실제 연결하거나 CPU fallback까지 넣는 구조 개선은 좋지만, 제출 직전에는 회귀 위험이 큽니다.

### 문제별 해결

#### 1) `chat_output.json` MOCK

원인: 이전 샘플/목업 출력이 루트 `outputs/chat_output.json`에 남아 있습니다.

해결:
```bash
bash chatbot.sh
```
완료 조건:
- `outputs/chat_output.json` 생성
- `[MOCK]` 0건
- 입력/출력 개수 일치
- `user`, `model` 키 존재

#### 2) ChromaDB 불일치

원인: `chunks.json`은 205건인데 기존 DB는 158건만 색인되어 있고 manifest도 없습니다.

해결:
```bash
python src/vectordb/build_db.py --fresh
python scripts/check_vector_db.py
```
완료 조건:
```text
[OK] 벡터 DB 정합성 확인: 205건
```

#### 3) 조교 노트북 로컬 실행 위험

원인: Qwen2.5-3B 4-bit, bitsandbytes, GPU 0 강제 배치 의존.

해결:
- README/발표자료에 **권장 환경: Colab T4 GPU** 명확히 작성
- 조교 로컬 노트북까지 보장하려면 `NO_LLM=1` 같은 fallback 모드 또는 CPU 모드 추가 필요

#### 4) Optional 미래 날짜 오류

원인: 게시일과 본문/마감/교육일 같은 미래 날짜가 섞이는 것으로 보입니다.

해결:
- 공지 handler에서 게시일 필드만 최신 기준으로 사용
- `date > today`는 최신 게시일 후보에서 제외
- 학사일정 질문은 `AcademicCalendarHandler` 우선 사용
- `realtime_output.json` 재생성 후 미래 날짜 검사

#### 5) 발표자료/영상/ZIP 부재

해결:
- 발표자료: 목표, Task1, Task2, RAG/라우터, UI, 실시간 기능, 한계 포함
- UI 영상: 질문 입력 → 답변 출력 → 대화 흐름 표시, 2분 내외
- 최종 파일명: `Termproject_박연진.zip`
