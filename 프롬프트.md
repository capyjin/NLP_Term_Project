# Task 1 프롬프트 — 로컬 제출 전 안전장치/문서 정리

아래 내용을 Claude에게 그대로 전달한다.

```text
너는 현재 자연어처리 Term Project 제출 직전 로컬 정리 담당자야.

목표:
GPU나 무거운 모델 실행 없이, 로컬에서 안전하게 할 수 있는 제출 전 안전장치와 문서 정리를 해줘. `최종정리.md`의 지적사항을 기준으로, 조교가 실행해야 할 파일과 실행 환경이 헷갈리지 않도록 README/가이드/검증 스크립트를 정리하는 것이 목표야.

현재 중요 근거:
- 과제 PDF상 조교는 `classifier_{이름}.ipynb`와 `chatbot.sh`를 실행한다.
- 현재 `classifier_박연진.ipynb`는 자동 루트 탐색이 들어가 있어 이전보다 개선됐다.
- 현재 `outputs/chat_output.json`은 전부 `[MOCK]`라 제출 전 갱신 필요하다.
- 현재 `chroma_db`는 `chunks.json`과 불일치한다. `scripts/check_vector_db.py` 결과 DB 158건 / chunks 205건, manifest 없음.
- Qwen2.5-3B 4-bit는 GPU 의존이 강하므로 권장 환경은 Colab T4로 명확히 써야 한다.

작업 범위:
1. `README.md`를 점검하고, 권장 실행 환경을 더 명확히 써줘.
   - 권장: Google Colab T4 GPU
   - Python 3.10.12 / torch 2.5.1
   - Task1: `classifier_박연진.ipynb`
   - Task2: `bash chatbot.sh`
   - Windows 로컬은 `chatbot.bat` 보조용이지만 GPU 없는 로컬 실행은 보장하지 않는다고 명시
2. README에 제출 전 필수 검증 명령을 추가해줘.
   - `python scripts/check_vector_db.py`
   - `python -m compileall -q src scripts tests`
   - `outputs/chat_output.json`에서 `[MOCK]` 0건 확인
3. 가능하면 `scripts/pre_submit_check.py` 같은 가벼운 검증 스크립트를 추가해줘.
   - GPU/모델 로드 금지
   - JSON 구조 검사
   - `outputs/chat_output.json`의 `[MOCK]` 검사
   - `outputs/cls_output.json`의 `question/label` 검사
   - `outputs/realtime_output.json`의 `user/model` 검사
   - `chroma_db` 정합성은 기존 `scripts/check_vector_db.py`를 호출하거나 안내만 해도 됨
4. Task1 분류기와 Task2 라우터 관계를 README에 솔직하게 정리해줘.
   - Task1은 학습 기반 분류기로 평가 JSON을 생성
   - Task2는 실서비스 안정성을 위해 키워드/핸들러 + RAG 라우터를 사용
   - 발표자료에서 이 점을 설명해야 한다고 안내
5. API가 존재한다면 간단한 실행/요청 예시를 README에 추가해줘. 단, 과제 핵심 실행은 `chatbot.sh`라고 명확히 구분해줘.

금지사항:
- 무거운 모델 로드하지 마.
- `bash chatbot.sh`를 실행하지 마.
- `chroma_db`를 삭제하거나 재구축하지 마.
- 원본 데이터 삭제하지 마.
- 제출 ZIP을 아직 만들지 마.

검증 기준:
- `python -m compileall -q src scripts tests`가 통과해야 함.
- 새로 만든 검증 스크립트가 있다면 실행 결과를 보여줘.
- 변경 파일 목록과 왜 바꿨는지 요약해줘.
- 남은 작업은 Task 2~5로 넘겨줘.
```

# Task 2 프롬프트 — 로컬 코드 품질 수정: Optional 날짜/공지 오류 중심

아래 내용을 Claude에게 그대로 전달한다.

```text
너는 현재 자연어처리 Term Project의 Optional 실시간 정보 정확성 개선 담당자야.

목표:
GPU 없이 로컬에서 가능한 범위 안에서 `outputs/realtime_output.json`의 미래 날짜/공지 오류를 줄여줘. 특히 “가장 최근 공지사항” 질문에서 `2026-12-31` 같은 미래 날짜가 최신 게시일로 나오는 문제를 해결하는 것이 핵심이야.

현재 문제:
- `outputs/realtime_output.json`에 “가장 최근 공지사항” 답변이 `2026-12-31`을 최신 게시일로 표시한다.
- 검토일은 2026-06-11이므로 미래 날짜를 최신 게시일로 판단하면 Optional 실시간성 점수에서 감점 위험이 크다.
- 학사일정 질문에 일반 공지/채용/행사가 섞이는 증거도 있다.

작업 범위:
1. 관련 파일을 먼저 읽고 원인을 파악해줘.
   - `src/realtime_model.py`
   - `src/chatbot_router.py`
   - `src/handlers/notice_handler.py`
   - `src/handlers/academic_calendar_handler.py`
   - `data/raw/academic_calendar.json`
   - `data/raw/all_docs.json` 또는 공지 raw 데이터
2. “최신 공지” 판단에서 미래 날짜를 제외하도록 수정해줘.
   - 기준일은 실행일 `date.today()` 사용
   - 게시일 후보가 오늘보다 미래면 최신 게시일 후보에서 제외
   - 단, 본문에 포함된 행사일/마감일과 게시일을 혼동하지 않도록 주의
3. “학사일정” 질문은 가능하면 `AcademicCalendarHandler`를 우선하도록 라우팅을 점검해줘.
4. `src/realtime_model.py`가 GPU 없이도 식단/셔틀/공지/학사일정 핸들러로 처리 가능한 질문은 Qwen을 로드하지 않도록 유지해줘.
5. 경량 검증을 추가하거나 실행해줘.
   - `python src/realtime_model.py --input data/test_realtime.json --output outputs/realtime_output.json`
   - 실행이 무거운 모델을 부르면 중단하고 원인을 설명
   - 출력 JSON 구조 `user/model` 확인
   - 답변 안에서 오늘보다 미래 날짜가 “가장 최근 게시일”로 쓰이지 않는지 확인

금지사항:
- Qwen2.5-3B 전체 로드를 요구하는 변경 금지.
- `chatbot.sh` 전체 실행 금지.
- 크롤링이 실패한다고 전체 기능을 깨지게 만들지 마.
- 임시로 답변을 하드코딩해서 테스트만 통과시키지 마. 일반화 가능한 날짜 필터를 넣어줘.

검증 기준:
- `python -m compileall -q src scripts tests` 통과
- 가능하면 `python src/realtime_model.py --input data/test_realtime.json --output outputs/realtime_output.json` 실행
- `outputs/realtime_output.json`이 `user/model` 구조 유지
- “가장 최근 공지”가 미래 날짜를 최신 게시일로 답하지 않음
- 변경 파일과 원인/해결 요약 보고
```
